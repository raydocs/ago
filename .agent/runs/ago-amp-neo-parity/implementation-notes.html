  • Do not perform an immediate all-repository rename from ClaudeX to Ago; create Ago-owned protocol boundaries first and migrate assets deliberately.

  • Start with a headless local daemon even though the first customer surface is a desktop app; this preserves the later runner/orb path.

Changes And Tradeoffs

  • Added a long-running full goal because Amp parity spans multiple independently verifiable phases.
  • The current Cloudflare thread app is treated as a reusable archive/sync asset, not yet as Ago's canonical writable control plane.
  • The public claudex-local-orchestrator remains a coexistence dependency and routing lab: reuse its model-role policy, gateway/usage parsing, doctor/smoke patterns, and bounded agent profiles, but not its Claude session state, usage JSONL, prompt-prefix compaction detection, or launchd process state as Ago authority.
  • The deployed ClaudeX Threads system and this workspace's thread-app, threadsync, threadgraph, threadread, and threadfind are suitable Phase 2/5 projections, ingest, graph, export, redaction, and retrieval assets. D1 remains a read/archive replica until it consumes Ago's canonical event protocol.
  • Runtime foundation research recommends an Ago-owned thin durable runtime with pinned Pi packages behind an adapter. OpenCode supplies optional diff/desktop patterns; Codex supplies protocol, sandbox, approval, and executor patterns. None of their session or transport schemas becomes Ago's canonical model.
  • The research worker's “do not recommend GLM” note is superseded by the user's explicit decision: GLM remains eligible for later held-out Ago routing evaluation.
  • The Neo UX audit freezes the local v1 loop as workspace/executor → durable thread → background multi-thread observation → queue/steer/interrupt → Git-backed Changes → Activity/archive. Desktop, CLI, web, and mobile must be renderers over shared commands/read models, never separate thread products.
  • Existing transcript-derived thread graph remains useful for migration and import, but Ago must own future thread state directly.
  • Chronicle evidence corrected delivery order: managed orbs precede generalized named runners; truth-preserving retrieval and orb OIDC are explicit parity phases before Agent-to-Agent.
  • Pi integration is conditionally approved only through a tiny Ago-owned TypeScript SDK sidecar pinned to earendil-works/pi commit 3da591ab…21 / packages 0.80.10. It must use SessionManager.inMemory(), noTools:"builtin", Ago-owned transcript/tool persistence, and exhaustive event normalization. Stock pi --mode rpc fails the authority boundary.
  • The deterministic Pi gate is now accepted: package integrities match the researched lock, public imports compile, all seven behavior tests execute, and the adapter creates no Pi JSONL authority. Promotion to a credentialed provider remains a separate smoke gate.
  • Trusted plugins run in one supervised Bun child per workspace/runtime generation over bidirectional versioned JSONL. This is a reload/failure boundary, not a security sandbox. Ago owns policy folding, tool execution, thread state, and rendering; built-in ago.permission.default is pinned last.
  • Automatic macOS tools must use a pipe-only broker and tiny per-job supervisor with a deny-default parameterized Seatbelt profile, exact derived environment, synthetic HOME/TMP, network disabled, bounded head/tail output, immutable launch digest, and process-group TERM→grace→KILL. Saved PIDs/PGIDs are audit-only and no broad unsandboxed fallback is allowed.
  • Daemon restart never blindly reruns an active turn with unknown side effects. Startup now records a visible turn.failed orphan-reconciliation event; graceful SIGTERM first persists cancellation, terminates the process group, and waits for cancellation settlement.
