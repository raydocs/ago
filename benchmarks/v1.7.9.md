# ClaudeX Flow v1.7.9 regression note

This is a **single same-task regression**, intended to catch workflow latency regressions. It is not evidence that one harness or model is generally faster.

| Field | v1.7.8 | v1.7.9 |
|---|---:|---:|
| Wall time | 134.44 s | 89.63 s |
| Quality | — | 31 / 31 checks |
| Diff hygiene | — | pass |
| Post-collect tail | 87.92 s | 38.36 s |

The observed wall-time reduction was 33.3%. v1.7.9 changed generic workflow mechanics: compact collect receipts, artifact-backed patches, deterministic whitespace repair, an integration acceptance gate, and a compact terminal outcome receipt.

## Controlled dimensions

- Same task and acceptance contract.
- Same project state and verifier.
- Same root/worker model policy.
- Same worker deadline and scope isolation.

## Validity limits

- One stochastic run per version.
- The row does not isolate model latency from harness latency.
- Only v1.7.9's full 31-check result and diff-hygiene result are asserted here.
- Results should be replicated across serial, parallel, research, and multi-module workloads before changing default admission thresholds.

The project policy therefore treats this row as a regression signal, not a winner declaration.
